# GA 实战会话沉淀：数据源与报告工作流

**沉淀时间**: 2026-05-25  
**来源**: 2026-05 上旬至 5 月 25 日 GA 股票分析会话、定时任务、报告生成与工具调试记录  
**定位**: 把“聊天里的临场经验”整理成可复用的交易研究流程。

---

## 一、核心原则

### 1. 真实数据优先
所有结论先问三件事：

1. **数据从哪里来**：行情、公告、新闻、板块涨跌、个股 K 线必须标注来源或采集路径。
2. **数据是什么时间的**：交易日、盘中、收盘后、周末发酵期的含义不同。
3. **数据能不能复现**：能落到脚本、数据库、CSV、报告文件的结论，优先级高于只在聊天里说一遍的观点。

> 实战心法：先拿真实数据，再讲故事；故事必须能被涨跌幅、成交额、封板质量、公告原文或事件时间表校验。

### 2. 轻量接口优先，全量扫描入库
会话中多次验证：全市场实时接口容易超时，单股、指数、板块等轻量接口更适合盘中临时分析；全量扫描更适合收盘后定时任务。

| 场景 | 推荐方式 | 使用原则 |
|------|----------|----------|
| 单只股票形态 | AKShare 个股历史/个股信息接口 | 快速验证 K 线、均线、成交量 |
| 指数和板块 | AKShare 指数/行业板块接口 | 判断市场环境和主线强弱 |
| 历史日线 | Baostock 或 AKShare 历史接口 | 适合回测和复盘 |
| 盘中高频/自动交易 | QMT / MiniQMT | 需要券商环境与权限，适合 Level-2 和下单 |
| 基本面深研 | 萝卜投研、理杏仁、公告网页 | 适合人工验证逻辑链 |
| 新闻与公告 | 财联社、东方财富、交易所公告、网页搜索 | 用于催化确认和时间表校验 |

### 3. 结论要分层
同一句“看好”必须拆成四层：

1. **事实层**：今天发生了什么，数据是多少。
2. **逻辑层**：为什么影响这个板块或个股。
3. **位置层**：K 线、量能、均线、板块梯队处在什么阶段。
4. **动作层**：低吸、持有、观察、放弃、止损分别对应什么条件。

## 二、每日工作流

### 盘前：事件与预案
- 扫周末/隔夜新闻、公告、政策、海外映射。
- 给事件分级：S 级必看、A 级关注、B 级观察。
- 先列方向，不先定股票：事件 -> 板块 -> 产业链环节 -> 个股池。
- 写出两种剧本：高开兑现、分歧低吸。

### 盘中：只验证，不幻想
- 先看指数环境：放量进攻、缩量震荡、放量下跌。
- 再看板块强度：涨幅、涨停家数、成交额、龙头封单。
- 最后看个股位置：是否在 5 日线/10 日线附近，是否缩量，是否有承接。
- 盘中临时筛选优先用轻量接口，避免全市场接口拖死分析节奏。

### 收盘后：复盘入库
- 获取当日行情和涨停数据，写入本地数据库或 CSV。
- 按板块归因：涨停原因、事件催化、龙头/中军/补涨梯队。
- 记录失败样本：炸板、高开低走、放量滞涨、跌破均线。
- 输出次日观察清单，而不是只输出当日情绪。

### 周末：方法论升级
- 把本周反复出现的判断模式写成 SOP。
- 把有效案例加入 `examples/` 或 `docs/`。
- 把脚本能自动化的部分沉淀到 `scanner.py`、`scanner_ma5_monitor.py` 或数据任务中。
- 把纯主观判断降级为“观察项”，直到有数据验证。

## 三、标准报告结构

正式周报使用 8 章结构，避免散点式输出：

1. **本周全景概览**：一句话总结 + 指数/成交额核心表。
2. **指数日线复盘**：逐日行情、关键转折和量能变化。
3. **板块结构分析**：领涨、领跌、抗跌、资金抱团方向。
4. **个股点睛**：龙头、中军、补涨、风险个股。
5. **重大事件复盘**：政策、产业、业绩、海外映射。
6. **资金流向分析**：成交额、ETF、融资、北向等可得数据。
7. **交易宪法视角**：因果性验证、题材阶段、卖点信号。
8. **下周展望**：关键节点、两种剧本、操作原则。

## 四、自动化沉淀清单

### 已经适合自动化的环节
- 收盘后抓取真实行情并入库。
- 涨停板块和个股梯队整理。
- “涨停/巨量阳线 + 回踩 5 日线 + 缩倍量”形态扫描。
- K 线图生成与买点标注。
- 周报 Markdown / HTML 归档。

### 暂不适合完全自动化的环节
- 事件真实性判断：需要人工或多源交叉验证。
- 个股强关联判断：需要公告、主营业务、订单、产业链位置确认。
- 盘中临场买卖：必须结合盘口、分时承接和个人风险承受能力。

---

## 五、风控底线

1. 不把未经验证的网络传闻当作交易依据。
2. 不把板块利好自动等同于个股利好。
3. 不在全市场数据源异常时强行输出“确定结论”。
4. 不因为某只股票符合形态，就忽略指数环境和板块退潮。
5. 所有报告仅供学习交流，不构成投资建议。
